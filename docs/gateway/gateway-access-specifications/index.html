接入指南

一.申请ClientId​

ClientId 申请说明​

ClientId 是三方资源调用网关的唯一标识，用于区分不同的客户端和调用方。在 GateWay-admin 网关平台中申请 ClientId。所有的 ClientId 需要妥善保管，避免泄露，以确保系统的安全性和稳定性。

您的经销商名称需要调用的服务名需要调用的URL列表计划上线日期备注
DR- Coffeesrv/v1/account/org/getToken
/v1/account/org/getUser		2020/4/20

申请流程​

  1. 准备信息：请确保您已准备好所需的经销商名称、服务名及相关URL。
  2. 下载模板：请点击这里下载申请模板。
  3. 填写表格：按照模板格式，整理并填写相关信息。
  4. 发送申请：将填写好的信息制作成 Excel 文档，并发送至 lang.hu@dr-coffee.com（IOT负责人账邮箱账号）。邮件主题请使用【网关调用申请-调用方名称】。请在邮件中附上以下内容：
    • 填写完整的申请表格
    • 相关的支持文件（如有）
    • 您也可以自己构建 Word 模板，但需要尽量符合模板格式。

二.获取Token​

二.获取Token​

获取Token流程​

  1. 生成签名：在发送请求之前，您需要根据申请到的 SecretKey 对请求进行签名。请参考签名规范文档以获取详细的签名方法和步骤。

  2. 发送请求：使用申请到的 ClientId 和经过签名的请求，向指定的 Token 生成接口发送请求。Token 需要在请求的 Header 中携带。获取 Token 的地址是 https://api.example.com/token/endpoint，这是用于身份验证的关键步骤。

    POST /token/endpoint
    Host: api.example.com
    Content-Type: application/json

    {
        "Client-ID": "您的 ClientId",
        "X-Sign": "生成的签名", // 使用 SecretKey 生成的签名
        "X-TimeStamp": "当前时间戳",
    }

  3. 请求参数：确保请求中包含以下参数：

    • Client-ID: 您的 ClientId
    • X-Sign: 使用 SecretKey 生成的签名
    • X-TimeStamp: 当前时间戳

  4. 接收响应：成功后，您将收到包含 Token 的响应。请妥善保存该 Token。获取 Token 是为了确保后续 API 调用的安全性和有效性。

  5. 使用 Token：在后续的 API 调用中，将 Token 与 ClientId 添加到请求头中，以进行身份验证。

    GET /api/endpoint
    Host: api.example.com
    Authorization: Bearer {您的 Token}
    Client-ID: {您的 ClientId}

注意事项​

  • 请确保请求参数的准确性，以避免请求失败。
  • SecretKey 不应直接传递给服务器，而是用于生成签名。请参考签名规范文档以获取详细信息。
  • Token 有效期通常为一定时间，请在过期前重新获取。
  • 请确保所提供的信息准确无误，以避免申请延误。
  • 申请后请耐心等待审核结果，通常会在3个工作日内回复。
  • 申请通过后会给您提供对应的 SecretKey 与 ClientId 用于验证及签名
