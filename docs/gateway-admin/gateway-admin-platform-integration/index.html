功能详情

概述​

本接入说明文档详细介绍了如何将普通 API 资源与第三方 API 资源接入网关管理平台。平台支持对接入的 API 进行灵活的流量控制、请求校验等高级配置，帮助您高效管理 API 资源，特别是对于第三方 API 的接入，平台提供了强大的规则配置能力。接入过程中的规则配置将决定哪些请求能够被允许，哪些请求会受到限流限制，如何确保请求携带特定的请求头、参数或请求体值等。

接入步骤​

步骤 1：登录平台并选择场景​

  1. 登录网关管理平台的管理后台。
  2. 在左侧导航栏选择 场景管理。
  3. 创建新的场景或选择已有场景，点击进入该场景的管理界面。

步骤 2：资源配置​

  1. 在场景管理页面，点击 资源管理。

  2. 点击 新增资源，填写资源相关信息：

    • 资源名称：输入您要接入的 API 名称。
    • 资源类型：选择资源类型（例如 REST API、WebSocket，第三方 API）。
    • 绑定场景：选择该资源所在的场景。
    • 资源详情：输入资源的详细描述或相关配置信息。

  3. 提交资源后，资源信息将通过平台的 Redis 推送机制实时更新到网关中。

步骤 3：规则配置​

在规则配置中，您可以为每个资源绑定特定的规则，并灵活配置请求头、请求参数（params）或请求体（body）等要求。不同类型的规则（如允许类、限流类等）可以针对这些维度进行具体的配置。根据规则类型，平台会强制要求请求携带特定的维度值（如请求头、请求参数、请求体）。

选择规则类型​

  1. 在 规则管理 页面，创建新的规则或选择已有规则进行修改。
  2. 选择规则类型：
    • 允许类规则：允许类规则用于检查和确保 API 请求必须符合指定的要求。例如，必须携带特定的请求头或请求参数，如果请求中没有携带这些内容，则请求会被拒绝。
    • 限流类规则：限流类规则用于对接入的请求进行流量控制，您可以配置请求头、请求参数或请求体中的具体值来对 API 流量进行限制。如果请求没有携带这些配置的值，系统会直接拒绝该请求。
    • 拒绝类规则：用于禁止不符合条件的请求，确保不合规的请求无法通过网关。
    • 频率类规则：用于限制单位时间内请求的次数，确保请求不超过设定的频率。

配置统计维度及维度来源​

在规则配置中，您需要配置请求必须携带的统计维度及维度来源。可以选择以下来源：

  • params：请求参数
  • body：请求体
  • header：请求头

每种规则类型都支持配置这些维度的值。例如，您可以为 允许类规则 配置以下内容：

  • 请求头 X-scene 的值为 gateway_v1。
  • 请求参数 user_id 的值为 12345。
  • 请求体字段 user_data 的值为 {"name": "John Doe"}。

配置允许类规则​

  1. 对于 允许类规则，您需要在规则中明确指定哪些维度（请求头、参数或请求体）需要被校验。例如：

    • 请求头校验：配置请求必须包含某个特定的请求头和对应的值。
    • 请求参数校验：配置请求必须包含某个特定的请求参数和对应的值。
    • 请求体校验：配置请求体必须包含某个特定的字段和对应的值。

  2. 示例配置：

    • 配置请求头 X-scene 的值必须为 gateway_v1。
    • 配置请求参数 user_id 必须为 12345。
    • 配置请求体 user_data 必须包含 {"name": "John Doe"}。

  3. 注意：如果绑定了该规则的资源请求没有携带这些配置的请求头、参数或请求体内容，网关将直接拒绝该请求，并返回错误信息。

配置限流类规则​

  1. 对于 限流类规则，您可以根据请求头、请求参数或请求体的特定值来限制流量。例如，您可以配置：

    • 请求头 X-scene 的值为 gateway_v1 时，限制每秒最多 100 次请求。
    • 请求参数 user_id 为 12345 时，限制每分钟最多 1000 次请求。

  2. 示例配置：

    • 配置请求头 X-scene 的值为 gateway_v1，并设定每秒最大请求数为 100。
    • 配置请求参数 user_id 为 12345，并设定每分钟最大请求数为 1000。

  3. 注意：如果请求中没有携带配置的请求头、参数或请求体内容，则会被直接拒绝。

绑定规则与资源​

  1. 完成规则配置后，您需要将规则绑定到具体的资源上。资源可以有多个规则绑定，每个规则独立存在，且不与场景绑定。规则的配置将根据资源的需求来确保 API 请求符合预期。
  2. 通过选择资源并将其与配置好的规则进行绑定，您可以为该资源配置多个规则类型，如允许类、限流类等，确保资源的流量控制、请求校验等能够符合您的需求。

步骤 4：审核与发布​

  1. 在审核页面查看待审核的配置项，确保所有绑定和规则设置符合要求。
  2. 审核通过后，规则与资源将通过 Redis 服务实时同步至网关，生效后立即开始处理流量。

步骤 5：监控与调整​

  1. 在 监控大盘 页面，您可以查看资源和规则的运行状态，包括流量、响应时间、成功率等。
  2. 如果发现配置不当，您可以快速调整规则设置，所有变更将通过 Redis 推送到网关，确保流量控制的即时生效。

平台配置说明​

资源配置​

资源是网关中的核心单元，您需要为每个 API 配置基本信息，并将其与相应的规则进行绑定。每个资源都可以有一个或多个规则配置，支持灵活的流量管理。特别是第三方 API 资源，您需要配置请求头和认证信息，确保 API 能够通过网关进行正确的路由和流量控制。

规则配置​

规则用于控制资源的行为，例如请求的允许、限流、请求参数的校验等。每个规则都具有独立的规则类型（如允许类、限流类等），并可绑定到多个资源上。每种规则类型会根据需求，灵活配置请求头、参数或请求体的校验。

请求头、参数与请求体校验​

您可以根据不同规则类型对请求进行校验：

  • 允许类规则：配置请求头、参数或请求体的校验，确保 API 请求符合预设要求。如果请求没有携带指定的值，系统将直接拒绝请求。
  • 限流类规则：根据请求头、请求参数或请求体中的具体值来限制流量，例如对某个特定的 API 或用户进行限流。如果请求没有携带这些配置的值，系统将直接拒绝该请求。
  • 拒绝类规则：用于禁止不符合条件的请求，帮助维护系统的安全性与稳定性。

常见问题与解答​

问题 1：如何为第三方 API 配置请求头？​

  • 在规则配置页面的 请求头校验 项中，您可以添加需要验证的请求头信息。为第三方 API 配置时，确保请求头名称和值与实际请求相匹配。

问题 2：如何查看规则是否生效？​

  • 在 监控大盘 中，您可以查看各资源的访问量、成功率等指标，实时跟踪规则的效果。

问题 3：我修改了规则，如何确保其实时生效？​

  • 修改后的规则将通过审核机制进入发布流程，审核通过后，规则将通过 Redis 推送到网关，立即生效。

文件命名规范​

资源、规则及场景的命名应该简洁、规范，确保统一管理。建议使用如下命名方式：

  • 资源：<场景名称>-<资源名称>
  • 规则：<规则类型>-<规则名称>
  • 场景：<场景类型>-<场景名称>
