Key-Auth 插件接入第三方的配置与执行流程

1. 整体流程概述​

在接入第三方时，整个流程包括获取 Token 和使用 Token 进行后续请求。以下是详细的执行步骤。

2. 执行流程​

2.1 第三方请求 Token​

  1. 第三方请求：

    • 第三方通过 clientId 和 secretKey 请求后端服务的 /getToken 接口，以获取访问 Token。
    • 请求示例：
    curl -X POST http://backend-service/getToken -d '{"clientId": "your_client_id", "secretKey": "your_secret_key"}'

  2. 后端服务验证：

    • 后端服务首先验证 clientId 和 secretKey 的有效性。此时，后端服务需要检查 secretKey 是否在配置中存在，以确保请求的合法性。
    • 这一步骤可以通过查找存储在数据库或配置文件中的消费者信息来实现，确保 clientId 和 secretKey 的组合是有效的。

  3. 生成 Token：

    • 如果验证成功，后端服务生成一个随机 Token，并将其与 clientId 关联存储，以便后续验证。
    • 返回示例：
    {
      "token": "your_generated_token"
    }

2.2 第三方使用 Token 进行后续请求​

  1. 携带 Token 发送请求：

    • 第三方在后续的 API 调用中，必须携带获取到的 Token，发送请求到网关。
    • 请求示例：
    curl -X GET http://gateway-service/api/resource -H 'Authorization: Bearer your_generated_token'

2.3 网关处理请求​

  1. 请求到达网关：

    • 网关接收到请求后，首先检查请求的 URL ，匹配到相应的资源。

  2. 启动 Key-Auth 插件：

    • 网关根据插件配置，决定当前资源是否需要启用 Key-Auth 插件。如果在 Key-Auth 插件中进行了配置，网关将调用 Key-Auth 插件的 Authenticate 方法。

  3. Key-Auth 认证：

    • Token 验证逻辑：

      • 插件从请求的 Header 中提取 API Key（Token）。通常，Token 会在 Authorization 头中以 Bearer 形式传递。
      • 提取示例：
      token := request.Header.Get("Authorization") // 获取 Token
      • 验证 Token：
        • 插件需要验证提取到的 Token 是否有效。为了实现这一点，插件可以查找存储的 Token 信息，确认该 Token 是否与之前生成的 Token 关联。
        • 具体步骤如下：
          • 使用 clientId 和提取到的 Token 查询存储的 Token 信息。
          • 如果找到匹配的记录，说明 Token 有效；如果没有找到，返回 401 错误，表示未授权。

  4. 规则引擎执行：

    • 如果 Token 验证通过，网关将继续执行规则引擎。规则引擎检查请求的 Header、Params 和 Body 是否符合预定义的规则。
    • 如果某个必需字段缺失，返回 400 错误，表示请求不合法。

  5. 请求转发：

    • 如果所有规则通过，网关将请求转发到后端服务。
    • 转发示例：
    response, err := http.Get("http://backend-service/api/resource", headers=request.Header)

2.4 Key-Auth 级别说明​

  • 网关级别：在网关层面进行 Token 验证，适用于所有请求。
  • 资源级别：在特定资源的路由中进行 Token 验证，允许更细粒度的控制。
  • 域名级别：根据不同的域名进行 Token 验证，适用于多租户场景。

3. 关于 getToken 接口的认证需求​

  • getToken 接口的认证：在这个流程中，getToken 接口需要通过 Key-Auth 插件进行认证。后端服务在验证 clientId 和 secretKey 的有效性时，确保 secretKey 在配置中存在是至关重要的。
  • 后续请求的认证：后续的 API 请求（如访问资源的请求）才需要携带 Token，并通过 Key-Auth 插件进行认证。

4. 总结​

通过以上配置和执行流程，您可以确保第三方能够安全地接入您的服务，并通过 Key-Auth 插件进行有效的认证和鉴权。确保在请求处理流程中，认证、资源匹配和规则执行的顺序合理，并且每个组件的配置清晰明了。
