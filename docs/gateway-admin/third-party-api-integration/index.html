三方接入

概述​

在网关管理平台接入第三方 API 时，您需要按照特定规则进行配置。当资源类型选择为 第三方 API 接入 时，平台将强制要求资源绑定特定的规则。该规则主要是为了验证请求头、请求参数或请求体中是否包含特定的值，以确保该 API 是合法的第三方接入。

本说明将详细介绍如何配置第三方 API 资源及相关规则，确保只有符合条件的请求才能通过网关。

第三方 API 接入步骤​

步骤 1：选择资源类型为第三方 API​

  1. 登录网关管理平台的管理后台。
  2. 在左侧导航栏选择 资源管理。
  3. 创建新资源时，选择 资源类型 为 第三方 API。
    • 资源名称：为您的第三方 API 设置名称。
    • 资源类型：选择 第三方 API。
    • 绑定场景：选择该资源所属的场景。
    • 资源详情：输入第三方 API 的描述、URL 或其他相关信息。

步骤 2：强制绑定内嵌规则​

当选择 第三方 API 类型时，平台会强制要求为该资源绑定一个 允许类规则。该规则将确保请求头中必须包含特定的请求头字段和对应的值。

配置允许类规则(该规则理论上是提前配置好的关于第三方API的一个强制性要求)​

  1. 在 规则管理 页面创建一个 允许类规则，配置请求头、请求参数或请求体中需要携带的特定字段。
  2. 在创建规则时，您可以配置以下项：
    • 请求头校验：配置请求头 X-scene 必须存在，并且其值必须为 third_party_api（表示该请求为第三方 API 请求）。
    • 请求参数校验（可选）：配置请求中必须包含的参数及其值，例如 api_key 必须为 your_api_key。
    • 请求体校验（可选）：配置请求体必须包含的字段及其值，例如 user_id 必须为 12345。

示例配置​

  • 请求头 X-scene 的值必须为 third_party_api，表示这是一个第三方接入的 API。
  • 请求参数 api_key 必须为 your_api_key，确保请求是合法的第三方 API 请求。
  • 请求体中的字段 user_id 必须为 12345，确保请求的数据来源正确。

配置规则后绑定资源​

  1. 规则配置完成后，必须将该规则绑定到 第三方 API 类型的资源上。
  2. 绑定规则后，只有携带符合要求的请求头、请求参数或请求体的 API 请求才能通过网关处理。任何缺少必要请求头或参数的请求将被拒绝。

强制规则说明​

当资源类型选择为 第三方 API 接入 时，平台会自动强制绑定该资源与一个 允许类规则，该规则用于验证请求的合法性。规则配置项如下：

  • 请求头校验：必须包含请求头 X-scene，并且其值必须为 third_party_api，表明该请求为合法的第三方 API 请求。
  • 请求参数校验（可选）：例如请求参数 api_key 必须为固定值 your_api_key。
  • 请求体校验（可选）：例如请求体字段 user_id 必须为 12345。

如果请求没有携带正确的请求头、请求参数或请求体字段，平台会拒绝该请求，并返回错误提示。

规则校验流程​

  1. 请求头校验：每次第三方 API 请求到达网关时，平台会首先检查请求头 X-scene 是否存在，并验证其值是否为 third_party_api。
  2. 请求参数校验：如果请求头通过校验，平台会接着检查请求参数（如 api_key）是否符合规定的值。
  3. 请求体校验：如果请求头和参数均通过校验，平台会进一步检查请求体中是否包含指定的字段（如 user_id）及其值。
  4. 请求通过与否：只有所有校验都通过的请求才会被网关接受。如果任何校验失败，系统会直接拒绝该请求，并返回相应的错误信息。

资源与规则绑定​

  1. 为了确保第三方 API 接入的合法性，必须为每个第三方资源绑定至少一个允许类规则。
  2. 绑定规则后，网关将自动对该资源的所有请求进行校验。资源可以绑定多个规则，确保在不同的情况下都能进行有效的请求验证。

配置注意事项​

  1. 强制绑定规则：当选择 第三方 API 接入 时，资源将自动绑定强制要求的规则，不能取消该绑定。
  2. 规则的配置内容：请确保规则中配置的请求头、请求参数或请求体的字段和值与实际请求一致，避免因不匹配导致请求被拒绝。
  3. 规则更新：如果您需要更改规则配置，可以在规则管理页面修改相关规则，并重新绑定到资源上。修改后的规则会立即生效。

常见问题与解答​

问题 1：为什么我的第三方 API 请求被拒绝？​

  • 如果您的请求缺少必须的请求头、请求参数或请求体字段，或者这些字段的值不符合规则配置，系统将拒绝该请求。请检查请求是否包含了正确的 X-scene 请求头、api_key 参数等。

问题 2：我可以为第三方 API 配置多个规则吗？​

  • 可以，您可以根据实际需求为第三方 API 配置多个规则。例如，可以同时配置请求头、请求参数和请求体的校验规则。

问题 3：如何修改第三方 API 的规则配置？​

  • 您可以在 规则管理 页面修改已绑定的规则。修改后，新的规则将自动生效，并应用于所有绑定该规则的资源。

总结​

通过上述步骤，您可以为网关中的 第三方 API 接入 配置强制规则，确保所有接入的第三方请求符合预定的安全和验证要求。通过这种方式，平台能够有效地保证第三方 API 请求的合法性和安全性。
